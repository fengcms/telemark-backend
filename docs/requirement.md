# 🎯 Telemark 系统：终极业务需求白皮书（复盘版）
我们这个系统的一切设计，都是为了服务好两个角色：作为管理者的你（和经理），以及在最前线打电话的电销员工。

## 💼 第一幕：你是如何管理和指挥全盘的？（管理后台的初心）
你开发管理后台，核心是为了解决三个老板最头疼的问题：钱花得值不值（线索质量）、线索怎么高效分发（防抢单内耗）、以及怎么看员工有没有偷懒（行为监控）。

### 1. 资产入库（买来数据怎么塞进去）
你从外面拿到了一个 Excel 表格（比如 5000 个号码）。你打开网页后台，一键上传。系统自动把这 5000 个号码打包成一个“特定批次”（比如“2026六月渠道A线索”），并记录你花了多少采购成本。系统在入库时会自动过滤掉库里已经存在的重复号码，保护公司不花冤枉钱。

### 2. 指挥分发（号码怎么变成员工的任务）
号码入库后是在“大公海”里。你勾选其中 1000 个，一键指派给“销售张三”。
安全防线一：张三只能在手机上看到这 1000 个号码，他看不到别人手里的，更没办法把这 1000 个号码打包下载带走。
安全防线二：系统在后台默默记账：“这 1000 个号是老板在几点几分分给张三的”。如果张三一直不打，你随时可以在后台收回。

### 3. 天眼监控（不听汇报，只看客观数据）
每天下班，你打开后台看板，不需要听员工编造口头汇报。你直接看自动生成的“员工日报”：张三今天早上几点打的第一通电话（有没有迟到）？下午几点打的最后一通（有没有早退）？他今天总共拨了多少次？真正接通了多少次？最硬核的指标：他今天纯通话时长是多少分钟？平均一通电话聊多久？（一眼抓出谁在敷衍，谁在认真沟通）。

## 📞 第二幕：一线员工是怎么用手机干活的？（员工 APP 的初心）
电销人员最讨厌繁琐的操作，也最容易产生疲劳感。员工 APP 的核心目标就是：消灭一切不必要的动作，让销售只专注于说话。

### 1. 无感登录与任务接收
销售早上打开手机 APP，登录账号。体验防线一：他只需要登录一次，之后哪怕关掉 APP、换了网络，系统都会通过“长短 Token”在后台悄悄帮他保持在线，绝对不会在他正要打电话时把他弹出来让他重新输密码。
APP 的界面极其简单：只有一个专属于他自己的客户列表（都是你分配给他的未拨打号码）。

### 2. 丝滑的外呼体验（消灭拨号盘）
销售看着列表，不需要用手指去按数字键，也不需要复制粘贴。他点击号码旁边的“拨打”图标，APP 瞬间自动唤醒手机自带的拨号盘，并把号码填好，他直接按绿色的呼叫键就能打出去。

### 3. 黄金 5 秒的强制闭环（消灭纸笔记录）
电话挂断，销售回到 APP 界面。这时候 APP 会立刻弹出一个不允许无视的“反馈弹窗”。
他只需要用手指点一下：如果是空号、或者是直接被挂断，他点一下“拒接”或“空号”，这个号码自动在列表里消失，彻底沉淀。
如果是聊得很好的意向客户，他点一下“已接听”，并在下面顺手打字或语音输入一行备注（如：“张总觉得价格还行，下周一回电”）。

系统副作用（数据全自动联动）：在销售点下“保存”的瞬间，系统自动在后台把这通电话的时间、时长、备注和结果上传给服务器。如果刚才那个电话是“已接听”，服务器会自动把这个号码的身份从“普通线索”升级为“意向客户”，不需要任何人手动去改。销售今天的个人战报和大盘数据，也会在这一瞬间自动在后台加 1，实时更新。
