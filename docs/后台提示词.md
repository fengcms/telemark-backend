一、 核心底层架构设计

1. 工业级无感刷新网络层：`@/api/client.ts`
   - 使用 Axios 建立一个统一的接口请求实例，基础路径(baseURL)指向 `http://localhost:8787`。
   - 编写响应拦截器，拦截 `401` (短 Token 过期) 错误。
   - 必须在文件顶部维护全局锁 `let isRefreshing = false;` 和挂起队列 `let failedQueue: any[] = [];`。
   - 当触发 401 时，第一个抢到锁的请求负责在后台调用 `/api/auth/refresh` 去续期（将浏览器本地的长 RefreshToken 传过去）。其他并发的 401 请求必须打包成 Promise 塞入 `failedQueue` 挂起排队。
   - 续期成功后，存入新短 Token，释放锁，并遍历执行队列重发失败的请求；续期失败（返回403）则彻底清空缓存，强行使用 `window.location.href = '/login'` 退回登录页。

2. 全局受保护路由守卫：`@/components/auth/ProtectedRoute.tsx`
   - 检查本地是否存在 AccessToken。若不存在，直接重定向到 `/login`。
   - 支持角色权限校验，若当前登录用户的 role 不符合页面路由权限，则重定向到大盘页或提示无权限。
